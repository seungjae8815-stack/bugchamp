# Play Console 인앱 상품 등록표 (자동 생성 — iap.json 기준)

> 수익 창출 → **제품 → 일회성 제품**(구 '인앱 상품') → **제품 만들기**. 전부 '구입(one-time)'.
> ⚠️ **제품 ID(=상품 ID) 는 한 글자도 다르면 안 된다.** 앱이 이 ID 로 상품을 조회한다
>    (`queryProductDetails`). 다르면 그 상품만 조용히 '판매 준비 중' 이 된다.
> ⚠️ 제품 ID 는 만든 뒤 **변경·재사용 불가**.

## 모든 제품 공통 설정
| 항목 | 값 |
|---|---|
| 구매 유형 | **구입**(one-time) |
| **구매 옵션 ID** | **`base`** — 구글 신규 필드. 밑줄(_) 불가, 하이픈만. 앱과 매칭 불필요(제품별 스코프라 전부 `base`). |
| 제품 세금 카테고리 | 디지털 앱 판매 |
| 결제 위치 제한 | 제한 없음 |
| 아이콘 | 선택(비워도 됨) |
| 태그 | 비워둠 |

## 소모성 / 비소모성

구글은 등록 시 이를 구분하지 않는다. 앱이 `consume` 하느냐로 갈리고,
그건 코드가 `iap.json` 의 `kind` 를 보고 처리한다. **전부 관리형으로 만들면 된다.**

| 반복 구매 가능(소모성) | 1회 한정(비소모성) |
|---|---|
| `jelly_s`, `jelly_m`, `jelly_l`, `jelly_xl`, `jelly_xxl` | `remove_ads`, `starter_pack`, `idle_pass`, `skin_gold_rhino`, `skin_albino_stag`, `theme_arena` |

## 등록 후 확인

- [ ] 11개 모두 **활성** 상태인지
- [ ] 앱 상점 화면에서 가격이 **콘솔 가격**으로 표시되는지(스토어 연결 확인)
- [ ] '개발 모드' 배너가 사라졌는지
